# 移动端适配指南

## 1. 已实现功能

### 1.1 响应式样式
- 自动检测屏幕尺寸，适配手机（≤768px）和平板（769-1024px）
- 侧边栏自动隐藏，转为底部导航栏
- 表格自动隐藏部分列，或转为卡片布局
- 表单元素自动调整布局

### 1.2 移动端组件
- `MobileLayout.vue` - 移动端布局组件（含底部导航）
- `useMobile.js` - 检测设备类型的 Composable

### 1.3 触摸优化
- 增大点击区域（适合手指操作）
- 优化 iOS 输入框缩放问题
- 添加触摸反馈效果

## 2. 使用方法

### 2.1 在页面中使用移动端布局

```vue
<template>
  <!-- PC 端使用原有布局 -->
  <div v-if="!isMobile" class="desktop-layout">
    <!-- 原有 PC 端代码 -->
  </div>
  
  <!-- 移动端使用 MobileLayout -->
  <MobileLayout v-else title="质量事件">
    <!-- 移动端内容 -->
    <div class="mobile-event-list">
      <div v-for="event in events" :key="event.id" class="event-card">
        <div class="event-title">{{ event.title }}</div>
        <div class="event-info">
          <span :class="'status-' + event.status">{{ event.status }}</span>
          <span>{{ event.due_date }}</span>
        </div>
      </div>
    </div>
  </MobileLayout>
</template>

<script setup>
import { useMobile } from '@/composables/useMobile'
import MobileLayout from '@/components/MobileLayout.vue'

const isMobile = useMobile()
</script>
```

### 2.2 隐藏 PC 端元素（移动端）

```vue
<!-- 在元素上添加 class，在移动端自动隐藏 -->
<el-table-column class-name="hide-on-mobile" />
```

### 2.3 移动端专用样式

```css
/* 在组件 style 中 */
@media screen and (max-width: 768px) {
  .my-component {
    padding: 10px;
  }
}
```

## 3. 页面适配建议

### 3.1 列表页
- PC：使用 `el-table`
- 移动端：使用卡片列表（`div.event-card`）

### 3.2 表单页
- PC：多列布局
- 移动端：单列布局，标签在上

### 3.3 详情页
- PC：左右分栏
- 移动端：上下堆叠

## 4. 测试方法

### 4.1 浏览器开发者工具
1. 按 F12 打开开发者工具
2. 点击设备切换按钮（或按 Ctrl+Shift+M）
3. 选择 iPhone、iPad 等预设设备
4. 刷新页面查看效果

### 4.2 真机测试
1. 确保手机和电脑在同一 WiFi
2. 启动前端开发服务器：`npm run dev`
3. 查看本机 IP：`ipconfig` 或 `ifconfig`
4. 手机浏览器访问：`http://本机IP:13000`

## 5. 优化建议

### 5.1 性能优化
- 移动端图片懒加载
- 减少不必要的请求
- 使用虚拟列表（长列表）

### 5.2 体验优化
- 添加下拉刷新
- 添加上拉加载更多
- 添加操作反馈（Toast）

### 5.3 PWA 支持（可选）
可配置为 PWA，支持：
- 添加到主屏幕
- 离线访问
- 推送通知

## 6. 已知问题

| 问题 | 解决方案 |
|------|---------|
| Element Plus 表格在移动端显示不全 | 使用卡片列表替代 |
| 弹窗在移动端过大 | 已添加响应式样式，宽度改为 95% |
| 底部导航遮挡内容 | 已添加 padding-bottom: 60px |
