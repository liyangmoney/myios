# 质量事件管理需求文档

**文档版本**: v1.0  
**编制日期**: 2026-03-02  
**编制人**: 系统开发团队  
**适用范围**: ISO 22163 质量管理体系信息化平台

---

## 1. 文档概述

### 1.1 目的
本文档定义了质量事件管理模块的功能需求，用于指导系统开发、测试和验收。

### 1.2 适用范围
适用于企业内部质量事件的创建、处理、跟踪和闭环管理，覆盖 PDCA 全生命周期。

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| PDCA | Plan(计划)-Do(执行)-Check(检查)-Act(处理) 质量改进循环 |
| 质量事件 | 需要按 PDCA 流程处理的质量相关问题或不符合项 |
| 当前处理人 | 当前阶段负责处理该事件的人员 |
| 通知人 | 需要接收事件进展通知的相关人员 |

---

## 2. 业务流程

### 2.1 事件生命周期

```
[新建] → [Plan计划] → [Do执行] → [Check检查] → [Act处理] → [关闭]
  ↑                                                    |
  └───────────────── [不通过退回] ←─────────────────────┘
```

### 2.2 状态流转规则

| 当前状态 | 操作 | 下一状态 | 说明 |
|---------|------|---------|------|
| NEW(新建) | 填写Plan | PLAN | 创建人指定Plan处理人 |
| PLAN | 填写Do | DO | Plan处理人指定Do处理人 |
| DO | 填写Check | CHECK | Do处理人指定Check处理人 |
| CHECK | 验证通过 | ACT | Check处理人指定Act处理人 |
| CHECK | 验证不通过 | DO | 退回Do阶段重新执行 |
| ACT | 关闭事件 | CLOSED | Act处理人关闭事件 |
| ACT | 保持打开 | ACT | 暂不关闭，保持跟踪 |

## 3. 功能需求

### 3.1 事件创建

#### 3.1.1 基本功能
- **FR-001**: 系统任何已登录用户均可创建质量事件
- **FR-002**: 创建人可删除自己创建的事件（删除前需确认）
- **FR-003**: 事件创建后自动发送邮件通知当前处理人

#### 3.1.2 事件基本信息
- **FR-004**: 事件编号：系统自动生成，格式为 `QE-YYYYMM-序号`
- **FR-005**: 事件标题：必填，最多500字符
- **FR-006**: 事件描述：选填，详细描述问题情况
- **FR-007**: 事件类型：单选，选项包括：
  - 内部不符合
  - 外部不符合
  - 审核发现
  - 过程异常
  - 设备异常
  - 其他
- **FR-008**: 严重度：单选，选项包括：
  - 轻微
  - 一般
  - 严重
  - 致命
- **FR-009**: 发生部门：选填，选择事件发生的部门
- **FR-010**: 截止日期：必填，日期选择器，格式 YYYY/MM/DD
- **FR-011**: 责任人：选填，指定事件责任人
- **FR-012**: 当前处理人：必填，指定当前阶段处理人
- **FR-013**: 通知人：可选多选，选择需要接收通知的人员列表

### 3.2 PDCA 各阶段处理

#### 3.2.1 Plan 阶段（计划）

**触发条件**: 事件状态为 NEW 或 PLAN，且当前用户为当前处理人

**可编辑内容**:
- **FR-014**: 根本原因分析：多行文本框，分析问题根本原因
- **FR-015**: 纠正措施计划：多行文本框，制定具体纠正措施
- **FR-016**: 附件上传：支持多文件上传（最多5个），单文件不超过10MB
  - 支持格式：图片(JPG/PNG/GIF)、PDF、Word、Excel
  - 文件显示：文件名、大小、下载链接
- **FR-017**: 指派下一步：选择 Do 阶段的处理人（必填）

**提交后动作**:
- 状态变更为 DO
- 当前处理人更新为指派的 Do 处理人
- 自动发送邮件通知新的当前处理人
- 记录操作日志，包含附件信息

#### 3.2.2 Do 阶段（执行）

**触发条件**: 事件状态为 DO，且当前用户为当前处理人

**可编辑内容**:
- **FR-018**: 实施过程记录：多行文本框，记录具体执行过程和结果
- **FR-019**: 附件上传：同 Plan 阶段
- **FR-020**: 指派下一步：选择 Check 阶段的验证人（必填）

**提交后动作**:
- 状态变更为 CHECK
- 当前处理人更新为指派的 Check 处理人
- 自动发送邮件通知
- 记录操作日志

#### 3.2.3 Check 阶段（检查）

**触发条件**: 事件状态为 CHECK，且当前用户为当前处理人

**可编辑内容**:
- **FR-021**: 验证结果：多行文本框，记录验证过程和结果
- **FR-022**: 附件上传：可上传验证相关的证据文件
- **FR-023**: 是否通过：单选，选项：
  - 通过，可以关闭 → 进入 Act 阶段
  - 不通过，需要重新处理 → 退回 Do 阶段
- **FR-024**: 指派下一步：根据选择指定 Act 处理人或退回 Do 处理人（必填）

**提交后动作（通过）**:
- 状态变更为 ACT
- 当前处理人更新为指派的 Act 处理人
- 记录验证人信息
- 发送邮件通知

**提交后动作（不通过）**:
- 状态退回 DO
- 当前处理人更新为指定的 Do 处理人
- 记录验证人信息
- 发送退回通知

#### 3.2.4 Act 阶段（处理）

**触发条件**: 事件状态为 ACT，且当前用户为当前处理人

**可编辑内容**:
- **FR-025**: 标准化措施：多行文本框，记录防止再发的标准化措施
- **FR-026**: 附件上传：可上传标准化相关文件
- **FR-027**: 状态选择：
  - 关闭事件 → 事件进入 CLOSED 状态
  - 保持打开 → 事件保持 ACT 状态继续跟踪

**提交后动作（关闭）**:
- 状态变更为 CLOSED
- 当前处理人设为空
- 记录关闭人信息
- 发送事件关闭通知给所有相关人员

### 3.3 附件管理

#### 3.3.1 上传功能
- **FR-028**: 各阶段在编辑对话框中提供附件上传功能
- **FR-029**: 支持拖拽上传和点击上传两种方式
- **FR-030**: 中文文件名正确处理，不乱码
- **FR-031**: 上传前文件大小检查，超过10MB提示错误
- **FR-032**: 上传成功后文件立即显示在列表中

#### 3.3.2 文件列表显示
- **FR-033**: 显示文件名、文件大小（自动转换 KB/MB）
- **FR-034**: 图片文件支持预览（点击缩略图放大查看）
- **FR-035**: 非图片文件点击下载
- **FR-036**: 编辑状态下可删除已上传的附件

#### 3.3.3 文件存储
- **FR-037**: 文件存储路径：`/uploads/quality-events/`
- **FR-038**: 文件命名：`时间戳_原始文件名.扩展名`
- **FR-039**: 提供文件下载接口：`/api/download?filename=xxx`

### 3.4 评论功能

- **FR-040**: 任何已登录用户均可发表评论
- **FR-041**: 评论内容支持多行文本
- **FR-042**: 评论支持附件上传，与 PDCA 附件管理一致
- **FR-043**: 评论按时间倒序显示（最新的在最上面）
- **FR-044**: 评论显示：用户名、发布时间、内容、附件列表

### 3.5 提醒功能

#### 3.5.1 邮件提醒
- **FR-045**: 事件创建时通知当前处理人
- **FR-046**: 状态变更时通知新的当前处理人
- **FR-047**: 事件关闭时通知所有相关人员（责任人、通知人列表）
- **FR-048**: 用户被创建时发送登录凭证邮件

#### 3.5.2 截止提醒
- **FR-049**: 事件截止前72小时开始发送提醒邮件
- **FR-050**: 提醒频率：每3小时发送一次
- **FR-051**: 提醒对象：当前处理人和通知人列表
- **FR-052**: 已关闭的事件不再发送提醒

### 3.6 操作日志

- **FR-053**: 自动记录所有操作：创建、更新、删除、评论
- **FR-054**: 日志内容显示：操作人、操作类型、时间
- **FR-055**: 更新操作详细显示变更内容：
  - 文本字段变更：显示字段名和新值
  - 状态变更：显示旧状态 → 新状态
  - 处理人变更：显示指派人信息
  - 附件上传：显示附件列表，文件名可点击下载
- **FR-056**: 评论操作：显示评论内容和附件
- **FR-057**: 日志按时间倒序排列

### 3.7 统计分析

- **FR-058**: 按状态统计：显示各状态事件数量
- **FR-059**: 按严重度统计：显示各严重度事件数量
- **FR-060**: 月度趋势：显示近6个月事件创建数量趋势
- **FR-061**: 统计结果以图表形式展示（柱状图、饼图、折线图）

### 3.8 列表与查询

#### 3.8.1 列表显示
- **FR-062**: 列表字段：事件编号、标题、类型、严重度、状态、当前处理人、截止日期
- **FR-063**: 逾期事件特殊标识（红色显示）
- **FR-064**: 分页显示，每页默认10条
- **FR-065**: 按创建时间倒序排列

#### 3.8.2 筛选功能
- **FR-066**: 按状态筛选
- **FR-067**: 按事件类型筛选
- **FR-068**: 按严重度筛选
- **FR-069**: "我的事件"筛选：显示当前用户是责任人/当前处理人/报告人的事件

### 3.9 权限控制

| 操作 | 权限要求 |
|------|---------|
| 创建事件 | 任何已登录用户 |
| 删除事件 | 创建人本人 |
| 编辑 Plan | 当前处理人 + 状态为 NEW/PLAN |
| 编辑 Do | 当前处理人 + 状态为 DO |
| 编辑 Check | 当前处理人 + 状态为 CHECK |
| 编辑 Act | 当前处理人 + 状态为 ACT |
| 发表评论 | 任何已登录用户 |
| 查看统计 | 任何已登录用户 |
| 查看日志 | 任何已登录用户 |

## 4. 非功能需求

### 4.1 性能需求
- **NFR-001**: 页面加载时间 < 3秒
- **NFR-002**: 文件上传支持最大 10MB
- **NFR-003**: 同时支持至少 100 个并发用户

### 4.2 安全需求
- **NFR-004**: 所有接口需 JWT 认证
- **NFR-005**: 密码使用 bcrypt 加密存储
- **NFR-006**: 文件下载需进行权限检查
- **NFR-007**: 防止 SQL 注入（使用参数化查询）
- **NFR-008**: 防止 XSS 攻击（前端转义输出）

### 4.3 可用性需求
- **NFR-009**: 系统可用性 ≥ 99.5%
- **NFR-010**: 支持主流浏览器：Chrome、Firefox、Edge、Safari
- **NFR-011**: 响应式设计，支持 1920x1080 及以上分辨率

### 4.4 数据备份
- **NFR-012**: 数据库每日自动备份
- **NFR-013**: 上传文件定期备份到远程存储

## 5. 界面需求

### 5.1 事件详情页布局

```
┌─────────────────────────────────────────────┐
│  基本信息（标题、类型、严重度、状态、截止日）    │
├─────────────────────────────────────────────┤
│  PDCA 流程图（可视化显示当前阶段）              │
├─────────────────────────────────────────────┤
│  Plan 阶段（根本原因、纠正措施、附件、填写人）  │
├─────────────────────────────────────────────┤
│  Do 阶段（实施记录、附件、填写人）              │
├─────────────────────────────────────────────┤
│  Check 阶段（验证结果、是否通过、附件、验证人） │
├─────────────────────────────────────────────┤
│  Act 阶段（标准化措施、附件、关闭人）           │
├─────────────────────────────────────────────┤
│  评论区域                                     │
├─────────────────────────────────────────────┤
│  操作日志                                     │
└─────────────────────────────────────────────┘
```

### 5.2 编辑对话框

- **UI-001**: 各阶段编辑使用对话框形式
- **UI-002**: 对话框标题显示阶段名称和说明
- **UI-003**: 表单字段根据阶段动态变化
- **UI-004**: 附件上传组件显示在表单底部
- **UI-005**: 提供"取消"和"保存"按钮

### 5.3 颜色规范

| 状态 | 颜色 |
|------|------|
| NEW(新建) | 红色 |
| PLAN(计划中) | 橙色 |
| DO(执行中) | 蓝色 |
| CHECK(验证中) | 青色 |
| ACT(处理中) | 紫色 |
| CLOSED(已关闭) | 绿色 |

| 严重度 | 颜色 |
|--------|------|
| 轻微 | 灰色 |
| 一般 | 黄色 |
| 严重 | 橙色 |
| 致命 | 红色 |
