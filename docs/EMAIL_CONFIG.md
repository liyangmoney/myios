# 163邮箱配置指南

## 管理员创建用户邮件通知功能

系统支持在管理员创建用户时，自动发送邮件通知给用户。

## 配置步骤

### 1. 复制环境配置文件

```bash
cd backend
cp .env.example .env
```

### 2. 配置163邮箱参数

编辑 `.env` 文件，填写你的163邮箱信息：

```env
# 163邮箱配置（邮件通知功能）
EMAIL_USER=your_email@163.com
EMAIL_PASS=your_auth_code
EMAIL_FROM_NAME=PIS系统
APP_URL=http://localhost:3000
```

### 3. 获取163邮箱授权码

1. 登录你的163邮箱网页版
2. 点击顶部「设置」→「POP3/SMTP/IMAP」
3. 开启「SMTP服务」
4. 按照提示获取**授权码**（不是登录密码！）

### 4. 重启后端服务

```bash
cd backend
npm restart
```

## 使用说明

### 创建用户

1. 管理员登录系统
2. 进入「用户管理」页面
3. 点击「新增用户」
4. 填写用户信息（**邮箱必填**）
5. 系统会自动：
   - 生成随机初始密码
   - 发送邮件通知到用户邮箱
   - 在对话框显示初始密码（可复制）

### 重置密码

1. 在用户列表中点击「重置密码」
2. 系统会生成新密码并发送邮件通知

## 邮件内容示例

邮件包含以下信息：
- 系统登录地址
- 用户名
- 初始密码（高亮显示）
- 所属部门
- 用户角色
- 温馨提示

## 故障排查

### 邮件发送失败

1. 检查 `.env` 文件中的邮箱配置
2. 确认使用的是**授权码**而非登录密码
3. 检查163邮箱是否开启了SMTP服务
4. 查看后端控制台日志

### 用户未收到邮件

1. 检查用户邮箱地址是否正确
2. 查看垃圾邮件文件夹
3. 检查163邮箱的发送限制

## 安全提示

1. **不要将 `.env` 文件提交到 Git**
2. 建议定期更换邮箱授权码
3. 使用专门的系统通知邮箱，避免使用个人邮箱
