# 数据备份与恢复指南

## 1. 概述

本系统提供以下备份方案：
- **方案A**: 宿主机定时任务备份（推荐）
- **方案B**: Docker 自动备份服务
- **方案C**: 手动备份

---

## 2. 方案A：宿主机定时任务备份（推荐）

### 2.1 配置备份脚本

1. **编辑备份脚本配置**
   ```bash
   cd /opt/MyIOS/scripts
   vim backup.sh
   ```

2. **修改以下配置**
   ```bash
   BACKUP_DIR="/opt/backups/pis-system"  # 备份存储位置
   DB_NAME="pis_system"
   DB_USER="root"
   DB_PASS="你的MySQL密码"               # 修改为实际密码
   UPLOAD_DIR="/opt/MyIOS/backend/uploads"
   ```

3. **添加执行权限**
   ```bash
   chmod +x /opt/MyIOS/scripts/backup.sh
   chmod +x /opt/MyIOS/scripts/restore.sh
   ```

### 2.2 设置定时任务

1. **编辑 crontab**
   ```bash
   sudo crontab -e
   ```

2. **添加定时任务**（每天凌晨2点自动备份）
   ```bash
   # PIS 系统自动备份 - 每天凌晨2点
   0 2 * * * /opt/MyIOS/scripts/backup.sh >> /var/log/pis-backup.log 2>>!
   ```

3. **查看定时任务**
   ```bash
   sudo crontab -l
   ```

4. **查看备份日志**
   ```bash
   tail -f /var/log/pis-backup.log
   ```

### 2.3 查看备份文件

```bash
# 列出所有备份
ls -lh /opt/backups/pis-system/

# 示例输出：
# db_20260303_020001.sql.gz       # 数据库备份
# uploads_20260303_020015.tar.gz  # 文件备份
```

## 3. 方案B：Docker 自动备份服务

### 3.1 修改 docker-compose.yml

将 `docker-compose.backup.yml` 内容合并到主文件：

```yaml
version: '3.3'

services:
  # ... 现有服务 ...

  # 自动备份服务
  backup:
    image: mcuadros/ofelia:latest
    container_name: pis-backup
    depends_on:
      - mysql
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./scripts:/scripts:ro
      - /opt/backups/pis-system:/backups
    command: daemon --docker
    labels:
      - "ofelia.job-run.backup.schedule=0 0 2 * * *"
      - "ofelia.job-run.backup.container=pis-mysql"
      - "ofelia.job-run.backup.command=/scripts/backup-docker.sh"

volumes:
  mysql_data:
```

### 3.2 启动服务

```bash
cd /opt/MyIOS
docker-compose up -d backup
```

## 4. 方案C：手动备份

### 4.1 手动备份数据库

```bash
# 备份命令
mysqldump -u root -p pis_system > backup_$(date +%Y%m%d).sql

# 压缩备份
gzip backup_$(date +%Y%m%d).sql
```

### 4.2 手动备份上传文件

```bash
# 打包上传目录
tar -czvf uploads_$(date +%Y%m%d).tar.gz /opt/MyIOS/backend/uploads
```

### 4.3 一键手动备份

```bash
cd /opt/MyIOS
./scripts/backup.sh
```

## 5. 数据恢复

### 5.1 使用恢复脚本（推荐）

```bash
cd /opt/MyIOS
./scripts/restore.sh
```

按提示选择要恢复的备份文件。

### 5.2 手动恢复数据库

```bash
# 1. 解压备份文件
gunzip db_20260303_020001.sql.gz

# 2. 恢复数据库
mysql -u root -p pis_system < db_20260303_020001.sql
```

### 5.3 手动恢复上传文件

```bash
# 1. 备份当前文件（可选）
mv /opt/MyIOS/backend/uploads /opt/MyIOS/backend/uploads_old

# 2. 解压恢复
tar -xzvf uploads_20260303_020015.tar.gz -C /opt/MyIOS/backend/
```

## 6. 备份策略建议

### 6.1 生产环境

| 备份类型 | 频率 | 保留时间 |
|---------|------|---------|
| 数据库 | 每日 | 30天 |
| 上传文件 | 每日 | 30天 |
| 全量备份 | 每周 | 90天 |

### 6.2 备份存储建议

1. **本地备份** + **远程备份**
   - 本地：快速恢复
   - 远程（OSS/S3）：防止服务器故障

2. **配置远程同步**（可选）
   ```bash
   # 同步到阿里云 OSS
   ossutil cp -r /opt/backups/pis-system oss://your-bucket/backups/
   
   # 或使用 rclone 同步到各种云存储
   rclone sync /opt/backups/pis-system remote:backups
   ```

## 7. 常见问题

### Q1: 备份失败怎么办？

检查日志：
```bash
cat /var/log/pis-backup.log
tail -20 /var/log/cron.log
```

常见原因：
- MySQL 密码错误
- 磁盘空间不足
- 权限问题

### Q2: 如何测试备份是否有效？

定期做恢复测试：
```bash
# 在测试服务器上恢复
mysql -u root -p pis_system_test < backup_file.sql
```

### Q3: 备份文件太大怎么办？

1. **增量备份**（仅备份变更）
2. **压缩备份**（已默认启用 gzip）
3. **定期清理**（保留最近30天）

### Q4: 如何备份到远程服务器？

使用 scp 或 rsync：
```bash
# 在备份脚本末尾添加
scp /opt/backups/pis-system/*.gz user@remote-server:/backups/
```

## 9. 总结

| 方案 | 适用场景 | 配置难度 | 可靠性 |
|------|---------|---------|--------|
| 宿主机定时任务 | 单服务器部署 | ⭐⭐ 简单 | ⭐⭐⭐⭐ 高 |
| Docker 服务 | Docker Compose 部署 | ⭐⭐⭐ 中等 | ⭐⭐⭐⭐ 高 |
| 手动备份 | 临时备份 | ⭐ 最简单 | ⭐⭐⭐ 中 |

**推荐**: 生产环境使用 **方案A（宿主机定时任务）**，同时配置远程备份。
