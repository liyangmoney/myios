# 质量事件监管操作手册

**文档版本**: v1.0  
**适用对象**: 质量管理人员、系统管理员、部门负责人  
**编制日期**: 2026-03-02

---

## 1. 概述

### 1.1 手册目的
本手册指导质量管理人员如何有效监管质量事件的处理流程，确保所有事件按照 PDCA 循环及时闭环。

### 1.2 监管职责
- **质量部门**: 统筹管理所有质量事件，监督处理进度
- **部门负责人**: 监管本部门相关事件的处理情况
- **系统管理员**: 维护系统配置，处理技术问题

### 1.3 关键指标
| 指标名称 | 目标值 | 说明 |
|---------|--------|------|
| 事件按时关闭率 | ≥95% | 在截止日期前关闭的事件比例 |
| PDCA 平均周期 | ≤30天 | 从创建到关闭的平均时间 |
| 逾期事件数 | 0 | 超过截止日期未关闭的事件 |
| 退回重做率 | ≤10% | Check 阶段不通过退回的比例 |

## 3. 事件监管流程

### 3.1 日常巡检（每日）

#### 步骤1：查看逾期事件
1. 点击导航栏 **"质量事件"**
2. 在筛选栏选择 **"状态：全部"** + **按截止日期排序**
3. 红色标记的为逾期事件，需重点关注

#### 步骤2：查看待处理事件
1. 点击 **"我的事件"** 筛选（查看自己是责任人的事件）
2. 或查看列表中 **状态为 DO/CHECK** 的事件
3. 确认各阶段处理人是否已及时处理

#### 步骤3：跟进逾期事件
对于逾期事件，按以下优先级处理：
1. **致命/严重**级别：立即电话联系当前处理人
2. **一般**级别：发送邮件催促，抄送部门负责人
3. **轻微**级别：系统内评论提醒

### 3.2 周度回顾（每周五）

#### 步骤1：生成周报表
1. 进入 **"质量事件"** → **"统计分析"**
2. 查看本周数据：
   - 新建事件数
   - 关闭事件数
   - 逾期事件数
   - 各阶段平均停留时间

#### 步骤2：分析趋势
1. 查看 **"月度趋势"** 图表
2. 关注事件数量是否有异常增长
3. 分析高频事件类型，识别系统性问题

#### 步骤3：部门例会
1. 将逾期事件清单发送给相关部门负责人
2. 确认下周重点跟进事项
3. 记录会议纪要

### 3.3 月度复盘（每月末）

#### 步骤1：生成月度报告
1. 导出本月所有事件数据
2. 统计关键指标：
   ```
   按时关闭率 = 按时关闭事件数 / 应关闭事件数 × 100%
   平均处理周期 = 所有关闭事件天数之和 / 关闭事件数
   ```

#### 步骤2：分析典型案例
1. 选择2-3个处理优秀的事件作为标杆
2. 选择2-3个逾期或退回的事件分析原因
3. 形成改进建议

#### 步骤3：更新管理要求
根据分析结果，可能需要：
- 调整某些类型事件的处理时限
- 增加培训内容
- 优化流程节点

## 4. 事件详情监管

### 4.1 如何查看事件详情

1. 在事件列表中点击 **事件编号** 或 **标题**
2. 进入详情页，查看完整信息：

```
┌──────────────────────────────────────────────────────────────┐
│ QE-202603-001: 供应商来料检验不合格                           │
│ 类型: 外部不符合  |  严重度: 严重  |  状态: 🔵 DO(执行中)       │
│ 截止日期: 2026/03/15 (还剩 5 天)                              │
├──────────────────────────────────────────────────────────────┤
│ 基本信息                                                     │
│ 责任人: 张三  |  当前处理人: 李四  |  报告人: 王五            │
│ 发生部门: 采购部  |  通知人: 赵六、孙七                      │
├──────────────────────────────────────────────────────────────┤
│ PDCA 流程                                                    │
│ ✅ Plan → 🔵 Do → ⏳ Check → ⏳ Act                          │
├──────────────────────────────────────────────────────────────┤
│ Plan 阶段（已完成）                                          │
│ 根本原因: 供应商工艺参数设置不当...                          │
│ 纠正措施: 要求供应商整改并提交8D报告...                      │
│ 附件: 8D报告.pdf [下载]                                      │
│ 填写人: 张三  |  填写时间: 2026/03/02                        │
├──────────────────────────────────────────────────────────────┤
│ Do 阶段（进行中）                                            │
│ 实施记录: 待填写                                             │
│ [处理人可点击"填写"按钮编辑]                                 │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 各阶段监管要点

#### Plan 阶段监管
| 检查项 | 合格标准 | 不合格处理 |
|--------|---------|-----------|
| 根本原因分析 | 使用5Why或鱼骨图，找到根因 | 退回补充分析 |
| 纠正措施 | 具体、可执行、有责任人 | 要求细化措施 |
| 完成时间 | 通常1-3个工作日内完成 | 催促并了解原因 |

#### Do 阶段监管
| 检查项 | 合格标准 | 不合格处理 |
|--------|---------|-----------|
| 实施记录 | 详细记录执行过程和结果 | 要求补充细节 |
| 证据附件 | 有照片、报告等证明材料 | 要求补充证据 |
| 完成时间 | 根据措施复杂度，通常3-7天 | 协调资源支持 |

#### Check 阶段监管
| 检查项 | 合格标准 | 不合格处理 |
|--------|---------|-----------|
| 验证结果 | 数据证明问题已解决 | 退回Do重新执行 |
| 有效性 | 问题未复发，指标达标 | 要求延长观察期 |

#### Act 阶段监管
| 检查项 | 合格标准 | 不合格处理 |
|--------|---------|-----------|
| 标准化 | 更新文件、培训相关人员 | 要求完善标准化 |
| 横向展开 | 类似问题已排查 | 要求扩大范围 |

### 4.3 如何催促处理人

#### 方式1：系统内评论
1. 在事件详情页底部 **"添加评论"**
2. 输入催促内容：
   ```
   @李四 该事件距离截止日期还有2天，请及时处理。
   如有困难请联系我协调资源。
   ```
3. 点击 **"发表评论"**

#### 方式2：邮件提醒
1. 查看事件详情中的 **"当前处理人"**
2. 发送邮件（系统会在截止前72小时自动提醒）

#### 方式3：升级处理
对于多次催促无响应的情况：
1. 将事件 **"指派给"** 更改为部门负责人
2. 在评论中说明升级原因
3. 线下沟通汇报

## 5. 统计分析使用

### 5.1 进入统计分析
导航：**质量事件** → **统计分析**

### 5.2 各图表解读

#### 按状态统计（饼图）
```
新建 5个     计划 3个     执行 8个     验证 4个     关闭 20个
```
- **监管要点**: 执行中事件占比不宜过高，说明处理效率需要提升

#### 按严重度统计（柱状图）
```
致命 ■ (1)    严重 ■■■ (5)    一般 ■■■■■■ (12)    轻微 ■■■■■■■■ (20)
```
- **监管要点**: 致命和严重事件必须有专人跟进，每日检查

#### 月度趋势（折线图）
```
事件数
 25│        ╭─╮
 20│    ╭───╯ ╰──╮
 15│╭───╯         ╰───
 10│
  5│
   └───────────────────
    1月  2月  3月  4月  5月  6月
```
- **监管要点**: 关注异常波动，3月出现高峰需调查原因

### 5.3 数据导出
1. 在事件列表页面，设置筛选条件
2. 点击 **"导出数据"** 按钮（如已开发）
3. 或使用浏览器开发者工具导出为 CSV

## 6. 逾期事件处理

### 6.1 逾期预警机制

| 剩余时间 | 预警级别 | 处理动作 |
|---------|---------|---------|
| 72小时 | 黄色预警 | 系统自动邮件提醒 |
| 24小时 | 橙色预警 | 质量专员电话提醒 |
| 已逾期 | 红色预警 | 升级至部门负责人 |

### 6.2 逾期事件处理流程

```
发现逾期事件
    ↓
联系当前处理人了解原因
    ↓
判断：是否可延期？
    ↓是              ↓否
调整截止日期      协调资源支持
    ↓                ↓
跟进至关闭      更换处理人
    ↓                ↓
记录延期原因    记录处理过程
```

### 6.3 延期审批
如需调整截止日期：
1. 在事件详情页评论说明：
   ```
   经与处理人沟通，因 xxx 原因需延期3天。
   新截止日期：2026/03/20
   审批人：质量经理
   ```
2. 线下完成审批流程
3. 系统内保持记录

## 7. 常见问题处理

### 7.1 事件长时间卡在某个阶段

**现象**: 事件在 DO 阶段停留超过10天

**处理步骤**:
1. 联系处理人了解障碍
2. 可能原因及对策：
   - **资源不足** → 协调其他人员支持
   - **措施不可行** → 退回Plan重新制定
   - **优先级冲突** → 调整事件优先级

### 7.2 Check 阶段反复退回

**现象**: 事件多次从 Check 退回 Do

**处理步骤**:
1. 分析验证不通过的根本原因
2. 组织专题讨论会
3. 可能措施：
   - 更换处理团队
   - 寻求外部专家支持
   - 升级为公司级项目

### 7.3 事件重复发生

**现象**: 类似事件多次创建

**处理步骤**:
1. 在 Act 阶段检查标准化措施是否落实
2. 检查相关文件是否更新
3. 检查培训是否到位
4. 必要时启动管理评审

### 7.4 处理人不配合

**现象**: 处理人不响应、不处理

**处理步骤**:
1. 通过系统评论、邮件、电话逐级升级
2. 联系其直接上级
3. 在绩效考核中体现
4. 极端情况更换处理人

## 8. 系统维护

### 8.1 用户管理
导航：**系统管理** → **用户管理**

#### 添加新用户
1. 点击 **"新建用户"**
2. 填写信息：
   - 用户名（登录账号）
   - 真实姓名
   - 邮箱（接收通知）
   - 部门
   - 角色（admin/user）
3. 设置初始密码
4. 系统自动发送通知邮件

#### 禁用离职员工
1. 找到用户，点击 **"编辑"**
2. 状态改为 **"禁用"**
3. 确保其名下无待处理事件，如有需转交

### 8.2 数据备份

#### 数据库备份
```bash
# 每周执行
mysqldump -u root -p pis_system > backup_$(date +%Y%m%d).sql
```

#### 文件备份
```bash
# 备份上传的文件
tar -czvf uploads_backup_$(date +%Y%m%d).tar.gz /path/to/uploads
```

### 8.3 系统配置

#### 邮件配置
如需修改邮件发送设置，联系系统管理员修改后端 `.env` 文件：
```env
EMAIL_USER=新邮箱@163.com
EMAIL_PASS=新授权码
```

#### 截止日期提醒设置
如需调整提醒时间，修改后端配置后重启服务。
